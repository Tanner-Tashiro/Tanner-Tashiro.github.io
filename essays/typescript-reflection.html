Introduction: A New Way to Code

TypeScript promised a way to write safer code while still working in the familiar JavaScript ecosystem.

Learning TypeScript: Discovering Types

One of the first lessons I learned was the value of explicit types. Declaring interfaces, optional properties, and union types initially seemed verbose, but I quickly saw how they prevent runtime errors. For example:


interface User {
  name: string;
  age?: number; // optional
}

function greet(user: User) {
  return `Hello, ${user.name}!`;
}

const alice: User = { name: "Alice" };
console.log(greet(alice));

In this snippet, TypeScript ensures that every user object has a name property. Forgetting it triggers a compile-time error rather than causing unexpected behavior at runtime. Even with prior ES6 knowledge, I discovered new idioms like type guards, enums, and advanced union types, which make code more predictable and maintainable.

TypeScript in Software Engineering

From a software engineering perspective, TypeScript is a powerful tool. It acts as a safety net for large projects, reducing bugs and improving collaboration. While it requires additional setup compared to plain JavaScript, the trade-off is worthwhile: more maintainable code, clearer APIs, and fewer runtime surprises.

Athletic Software Engineering: Practicing the WODs

The practice “Workout of the Day” (WODs) pushed me to actively code under time constraints. Initially stressful, this style quickly became enjoyable as I noticed improvements in both speed and accuracy. Repetition, incremental difficulty, and immediate feedback make this “athletic” approach to learning highly effective for mastering TypeScript.
